# Memos Plus｜Obsidian 本地收集与投递工作台

Memos Plus 是一个面向 Obsidian 的本地优先收集与投递插件。

它的核心目标不是再做一个独立 memo 列表，而是把文字、链接、图片、任务、病例经验、项目想法和临时素材快速收进来，并在记录的同时发送到正确的文件、标题、模板或任务流程里。

一句话说：**不只是收集，而是让收集从一开始就有归处。**

## 插件定位

很多 Obsidian 用户遇到的问题不是不会写笔记，而是临时内容太多：

- 看到一段资料，想先收下来。
- 复制了一个链接，希望保留标题和来源。
- 想到一个项目需求，应该追加到项目文档的某个标题下。
- 记录一个病例经验，应该放到病例文件或疾病专题里。
- 手机上突然想到一件事，只想快速保存，之后再整理。

如果每次都要手动打开目标文件、搜索标题、调整格式、粘贴内容，收集就会变慢，素材也容易堆成新的收集箱。

Memos Plus 解决的是这条链路：**快速输入 -> 判断内容类型 -> 找到目标 -> 插入到正确位置 -> 稍后继续整理。**

## 核心特点

### 1. 本地优先，数据仍是 Markdown

Memos Plus 默认把 memo 写入 vault 内的年度 Markdown 文件，例如：

```text
我的资源/Memos/2026.md
```

这些内容不是封闭数据库。你可以直接打开、搜索、同步、备份和编辑。

插件不需要账号，不包含遥测、广告或云端依赖。它只读写你在设置里指定的 Markdown 文件和附件目录。

### 2. 多入口快速输入

Memos Plus 提供多个输入入口，适合不同场景：

- **主页面输入框**：适合连续记录和集中整理。
- **快速记录弹窗**：适合从命令面板或快捷入口快速打开。
- **右侧栏快速输入**：适合常驻在 Obsidian 侧边，不离开当前笔记也能记录。
- **移动端轻量入口**：适合 iPhone 上快速收集，不被复杂界面拖慢。

输入框支持常见素材类型和写作辅助：

- 普通文字
- 链接
- 图片
- 标签
- 待办任务
- 表格
- Callout
- 代码块
- Excalidraw 新建绘图
- 自定义发送格式

快速记录还可以自动带入当前选中文字，或按设置检测剪贴板中的文本、链接和图片。近期版本还加入了剪贴板去重，避免同一内容在主页、侧边栏、移动端或快速记录中被反复自动填入。

### 3. 发送到真实文件和真实标题

Memos Plus 的重点不是“保存一条 memo”这么简单。

你可以把当前输入发送到：

- 默认 memo 文件
- 项目文件
- 标签筛选出来的文件
- 最近使用的目标文件
- 搜索到的 Markdown 文件
- 目标文件里的真实 Markdown 标题
- 文件开头或文件末尾
- 新建标题之后的位置

比如：

- 把一个插件想法发送到项目文档的“需求池”。
- 把医学资料发送到疾病专题的“参考资料”。
- 把病例经验发送到病例文件的“复盘”。
- 把一个链接发送到主题笔记的“延伸阅读”。

这让 Memos Plus 更像 Obsidian 的内容投递层：内容进入插件之后，可以继续向知识库内部流动。

### 4. 发送格式规则，让内容按用途落地

同一段输入在不同场景下可能需要不同格式。

Memos Plus 支持把内容保存成：

- 普通笔记
- 链接资料
- Obsidian Tasks 任务
- Callout 折叠块
- 代码块
- 自定义 Markdown 模板

发送规则可以决定内容发到哪里、插到哪里、变成什么格式，以及发送成功后是否清空输入、保留、归档或删除原 memo。

近期版本还加入了“绑定标题”能力：当你发送到某个文件并选择匹配标题时，可以自动套用对应规则。例如选择“待办”标题时，自动弹出任务设置，并按任务格式写入。

### 5. 新建文件模板库

当目标文件还不存在时，Memos Plus 不要求你退出当前流程去手动建文件。

你可以直接在发送流程里使用模板创建新文件：

- 项目模板
- 疾病模板
- 病例模板
- 文献模板
- 药物模板
- 检查模板
- 软件开发模板
- 自定义模板分组

创建完成后，插件会继续进入标题选择和写入流程，把当前内容插入到新文件中。

这适合资料型、项目型、医学型 vault：搜索不到目标时，不是中断，而是顺手创建。

### 6. 与 Obsidian Tasks 配合

Memos Plus 可以生成 Obsidian Tasks 兼容的任务格式，并支持：

- 优先级
- 开始日期
- 计划日期
- 截止日期
- 创建日期
- 重复规则
- 任务正文详情

任务既可以由用户主动点击任务按钮创建，也可以由发送规则自动触发。

左侧的任务管理入口还可以按优先级和日期筛选任务，例如：

- 最高优先级
- 高优先级
- 逾期任务
- 今日到期
- 本周到期

开启全库任务筛选后，插件可以通过任务索引缓存从整个 vault 中筛选未完成任务，减少每次点击时的重复扫描。

### 7. 整理目录和筛选视图

Memos Plus 提供面向整理的导航入口：

- 全部笔记
- 项目目录
- 自定义目录
- 标签筛选
- 待整理
- 今日新增
- 未归档
- 有链接
- 有图片
- 未完成任务

这些入口让收集箱不只是堆放内容，而是可以变成一个待处理队列。

你可以先快速收集，再定期打开“待整理”“有链接”“未完成任务”等视图，把内容继续转入正式知识结构。

### 8. 移动端优先的稳定体验

Memos Plus 明确支持移动端，而不是把桌面弹窗原样搬到手机上。

移动端可以使用轻量首页，只保留最重要的输入和整理入口。发送到文件或新建文件时，移动端使用更稳定的页面式选择流程，减少多层弹窗、键盘遮挡和点击错位。

移动端设计重点是：

- 打开快
- 输入快
- 选择目标稳定
- 发送后不丢内容
- 复杂整理可以稍后回到桌面继续

## 适合谁

Memos Plus 适合这些 Obsidian 用户：

- 经常收集灵感、链接、图片和资料。
- 有大量项目笔记，需要快速追加想法和任务。
- 用 Obsidian 管理医学资料、病例经验或专题知识。
- 使用 Obsidian Tasks，希望快速创建结构化任务。
- 需要在手机端快速记录，同时保持桌面端整理能力。
- 希望数据保留在本地 Markdown 中，而不是进入封闭数据库。

## 推荐使用方式

刚开始可以只用最简单的流程：

1. 打开 Memos Plus。
2. 输入一条内容。
3. 默认保存到 memo 文件。
4. 定期查看“待整理”。
5. 对重要内容使用“发送到文件”或“转到项目”。

熟悉之后，再逐步配置：

- 常用项目文件
- 标签筛选页
- 发送格式规则
- 新建文件模板库
- 任务管理
- 移动端轻量首页
- 侧边栏快速输入

## 总结

Memos Plus 的特点可以概括为四句话：

- 它是本地优先的，内容仍然保存在 Obsidian Markdown 文件里。
- 它不只是 memo 列表，而是收集、投递、整理一体的工作台。
- 它能把内容发送到真实文件、真实标题、模板新文件和任务格式中。
- 它兼顾桌面端深度整理和移动端快速记录。

对经常在 Obsidian 中处理项目、资料、任务、病例和临时想法的人来说，Memos Plus 可以把很多分散操作压缩成一个更顺手的内容流转流程。
