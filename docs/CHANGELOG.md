# Changelog

## 0.1.60 - Unreleased

- 新建文件模板库支持第三方 Templater 渲染：选择模板创建文件时，如果 `templater-obsidian` 已安装且启用，会优先用 Templater 的运行时解析 `<% ... %>` / `<%* ... %>`，并为 `tp.file.title`、`tp.file.creation_date()` 等函数提供真实目标文件上下文；Templater 不可用或执行失败时会提示并回退到 Memos Plus 现有 `{{title}}`、`{{content}}`、`{{tag}}`、`{{source}}`、`{{folder}}`、`{{date}}`、`{{time}}` 变量替换。
- 输入框架构做最小稳定性收口：主页、移动端主页、侧边栏和快速记录弹窗继续共用现有 `createComposerSession → ComposerWidget → createNativeMarkdownComposer` 链路，但现在显式传入 `home` / `mobileHome` / `sidebar` / `quickCaptureModal` surface，移动端键盘避让不再靠猜测容器来选择策略。
- 抽出共享 `TaskOptionsForm`：输入框任务按钮和“发送到”弹窗任务选项现在共用同一套优先级、开始日期、计划日期、截止日期、完成日期、重复和创建日期开关表单，并继续复用原有任务 Markdown 生成逻辑，避免不同入口出现 `- - [ ]`、`- * [ ]` 等重复任务前缀。
- 将红色“侧边栏启动设置”同步放进“界面布局 > 侧边栏 > 快速输入框”的右侧区域设置顶部：点击预览里的快速输入框后也能直接看到“启用侧边栏快速输入”和“自动打开侧边栏”，不再只依赖侧边栏布局页顶部入口。
- 发送到弹窗的“搜索”页底部新增常驻“新建文件”操作：搜索词不为空时显示 `新建“搜索词”`，即使已有匹配结果也可直接用当前搜索词进入现有新建文件模板库流程；顶部标签栏仍只保留搜索、项目、标签文件、最近和加号，不新增新建文件标签页。
- 将“侧边栏启动设置”放到“界面布局 > 侧边栏”顶部：新增红色醒目控制卡片，直接显示“启用侧边栏快速输入”和“自动打开侧边栏”两个开关，避免用户需要滚到页面下方寻找启动选项。
- 修复快速输入框聚焦时左侧多出紫色竖线的问题：保留外层输入框的单一紫色 focus 边框，移除 composer 聚焦时额外叠加的 1px/4px 紫色 shadow、内部原生 Markdown 编辑器和 textarea 回退输入层的重复 inset focus 阴影，并将 composer 内部输入光标改为普通文本色，避免空输入框光标看起来像贴边紫色边框。
- 收口移动端弹窗关闭后的键盘/重载风险：任务设置弹窗取消或确认后不再在 iPhone 上自动聚焦主页输入框，发送到项目的空内容提示和发送失败草稿保护也只在桌面端恢复焦点，避免弹窗关闭后触发 focus → visualViewport resize → scroll/render 的循环；发送弹窗内新增/重命名标签这类轻量输入在移动端改用系统 prompt，避免在 ProjectSendModal 上继续叠加 ProjectTagTabModal。
- 修复移动端原生 Markdown 快速输入框文字被压成竖排的问题：为 Memos Plus 自己的 CodeMirror composer 容器、内容层和行补齐宽度、最小宽度和横排书写约束，避免 iPhone/Obsidian Mobile 上聚焦输入后输入列宽坍缩成一个字符。
- 收口移动端刷新风暴：TaskIndex 文件变化不再无条件触发全局 `refreshViews()`，没有 Memos 主视图时直接跳过，有主视图时按移动端节奏防抖合并刷新，避免 Obsidian Sync 或批量文件变化造成短时间上百次视图刷新；剪贴板内容确认弹窗的“填入/追加/忽略”也接入移动端重复点击锁，并增强 unhandled rejection 日志栈信息。
- 扩展移动端闪退/重载诊断：诊断日志现在保留最近 200 条结构化事件，覆盖 load/save、主视图/侧栏渲染起止、弹窗选项点击、window resize、visualViewport、输入焦点和错误事件，并新增“导出 Memos Plus 调试日志”命令、设置页入口和移动端工具栏入口写入 `Memos Plus Debug Log.md`，方便在 iPhone 镜像复现后判断是插件重载、视图重建还是 WebView 中断。
- 快速输入框新增“清空输入内容”按钮：主页、侧边栏、移动端和快速记录弹窗共用同一入口；短内容可直接清空，长内容、链接、任务或附件引用会先确认。清空只局部重置输入框、Callout/任务临时状态和草稿缓存，不改变默认发送规则或当前发送目标，移动端点击不会主动聚焦输入框或触发整页重绘。
- 修复移动端主页快速输入框聚焦后的布局异常：主页输入框在 iPhone/Obsidian Mobile 上改为受控的 120px 起始高度和 40vh/280px 最大高度，textarea 回退不再信任异常 `scrollHeight` 把空输入框撑成大黑框；键盘弹出时只调整主页底部避让和输入框最大高度，工具栏与发送按钮保持两列布局，悬浮记录按钮会在输入框聚焦期间隐藏，避免遮挡“发送”按钮。
- 增加移动端弹窗稳定性诊断与安全收口：移动端默认输出带 `sessionId` 的 onload/onunload、视图、弹窗、设置保存、layout-change、visualViewport、focus/blur、window error 和 unhandledrejection 日志，用于区分真实插件重载、视图重建和 WebView 重启；弹窗关键按钮加入移动端重复点击锁，关闭后清理大型结果缓存并阻止异步结果继续渲染，Saved Search 移动端不再实时跑复杂预览，发送/模板列表移动端首批限制为 30 条，原生 Markdown 输入框移动端不再同时绑定 touch 与 click 聚焦。
- 修复剪贴板检测弹窗“填入/追加”未触发链接分析的问题：快速输入框手动粘贴、剪贴板弹窗填入和追加现在共用同一套输入内容处理入口，继续复用原有微信、哔哩哔哩、小红书、抖音和普通网页链接分析能力；追加内容会保留换行分隔，移动端弹窗应用内容时不再强制唤起输入焦点。
- 修复设置页顶部横向标签栏滚动位置丢失：点击“显示设置”“高级设置”等后方标签时不再重建顶部标签栏，切换内容后会恢复原有横向滚动位置并保持当前标签可见；“界面布局”现在位于标签栏最前面，并作为首次打开设置页的默认分类。
- 稳定移动端弹窗交互：快速记录、编辑、图标选择、保存搜索、发送标签/模板/路径选择等弹窗打开时不再在移动端自动聚焦输入框，避免弹窗动画、键盘唤起和视口滚动修正叠加导致 Obsidian Mobile 卡死或闪退；桌面端保留自动聚焦体验。
- 激进删除旧项目插入模板兼容层：移除 `legacyProjectTemplate.ts`、旧项目插入模板测试、`projectInsertTemplate` / `projectTemplateOptions` / `projectInsertHeading` / `createProjectHeadingIfMissing` 设置字段和归一化读取；默认“发送到项目”规则现在只由现代发送规则 `ManagedTemplate` 创建。
- 清理旧项目插入模板的可见文案：移除设置 i18n 中不再使用的 `projectTemplate.*` 和旧项目插入模板标签，避免“项目插入模板”历史概念重新出现在设置页。
- 收口旧项目插入模板的运行时回退：`store.ts` 写入项目/文件时不再读取旧模板字符串，未传入发送规则的旧调用会使用内置普通笔记格式兜底。
- 清理旧项目设置 UI 死代码：移除不再暴露的 `renderProjectSettings()` 和旧项目插入模板编辑器，避免旧入口被误接回设置页。
- 继续清理旧移动端布局路径：移除已不被主渲染使用的旧 `resolveMobileHomeModules` 导出，旧移动端布局 helper 只保留旧设置归一化和迁移用途，避免后续误接回主界面。
- 收口旧移动端布局设置：旧 `mobileHomeLayout` / `mobileHomeCustomModules` 继续用于兼容迁移，但设置 UI 不再显示旧下拉和旧模块开关，移动端显示内容统一从“界面布局”的 `mobileLayout` 配置入口管理。
- 继续减少发送弹窗重复读取：选择目标文件后读取到的 Markdown 标题会在当前弹窗生命周期内缓存，返回再进入同一文件时不再重复解析 headings。
- 继续收口发送弹窗的按需缓存：标签文件、自定义标签页和搜索页现在会在弹窗生命周期内缓存同一标签或同一搜索词的文件结果，重复切换标签页或返回搜索结果时不再重复触发 Vault 索引查询。
- 侧边栏快速输入现在默认随插件加载自动打开：插件会等待 Obsidian layout-ready 后复用已有侧边栏或创建一个新侧边栏，并且自动打开不会聚焦输入框、不会弹出移动端键盘；设置页新增“自动打开侧边栏”开关，可关闭该行为。
- 继续按需化发送弹窗数据加载：项目列表和最近文件不再在打开“发送到项目/文件”流程前预加载，只有切到“项目”或“最近”标签页时才读取并缓存结果，减少发送弹窗唤醒全库 `VaultMetadataIndex` 的机会。
- 补齐显示模块的 Vault metadata 需求声明：项目目录、项目筛选器和标签筛选现在会在统一 `DisplayModule` 注册表中标记 `vaultIndex` 数据需求，后续发送弹窗、标签文件、全库搜索和模板库可以继续沿用同一套按需加载判断；当前仍保持延迟搜索，不会在主界面初始渲染时预热全库索引。
- 继续收口显示模块的数据加载：统计卡片会显式声明需要 memo 数据，任务索引后台构建会先检查主页、侧边栏和移动端布局中是否存在任务目录；隐藏任务目录时，文件变化只标记任务缓存需要更新，不再立即后台扫描全库任务。
- 统一显示模块系统新增数据需求解析：`DisplayModule` 现在可以从当前主页/侧边栏/移动端布局汇总所需数据，主视图 `reload()` 会先判断当前布局是否需要 memo 数据；极简输入等不显示列表、数量、目录或统计的布局不再预先读取 memo 文档，为后续按模块加载 TaskIndex、VaultIndex 和统计数据打底。
- 收口侧栏目录模块判断：主页侧栏、移动端轻量侧栏和右侧栏快速输入目录现在共用 `displayModules` 中的目录模块 helper，避免 `allNotes / projectDirectory / taskDirectory / fileList` 等模块列表在多个文件里重复维护。
- 收口主页和移动端列表的结果区渲染：文件数量和 memo 列表现在共用 `renderHomeResults`，关闭文件列表模块时不会创建 timeline，也不会加载列表渲染。
- 收口主页和移动端列表的重复工具栏渲染：搜索框、Memos 设置按钮和刷新按钮现在共用同一个内部 helper，避免后续模块显示逻辑在桌面主页和移动端列表中重复维护。
- 统一显示模块系统继续落地到桌面主页：主页现在读取 `homeLayout` / `DisplayModule` 决定是否渲染侧栏、搜索框、设置按钮、刷新按钮、输入框、文件数量和列表；隐藏模块会跳过对应 DOM、事件绑定和列表加载。
- 统一显示模块系统继续落地到移动端轻量首页：移动端主页现在直接读取 `mobileLayout` / `DisplayModule`，不再通过旧 `mobileHomeLayout` resolver 决定渲染内容；隐藏输入框、目录、统计、热力图或文件列表时，会跳过对应渲染和列表加载。
- 重做“界面布局”的图形化预览：移除灰色模块长条列表，改为桌面主页、侧边栏和移动端的真实界面缩略图；点击输入区、目录区、任务区、文件列表等真实区域后，右侧只显示对应区域的常用设置，并保留“隐藏此区域 / 恢复默认 / 打开完整设置”。
- 统一显示模块系统继续落地到侧边栏快速输入：`homeLayout` / `sidebarLayout` / `mobileLayout` 现在保存显示顺序和紧凑模式，右侧栏会按 `sidebarLayout` 决定是否渲染快速输入框、目录、数量和结果列表；隐藏数量或列表时不再计算对应数据。
- 梳理重复设置的兼容关系：`mobileLayout` 作为新的移动端布局主字段，旧 `mobileHomeLayout` 仅在缺少新字段时迁移；发送规则新增“跟随全局 / 自定义”用于清空输入框和转出后处理，任务页只显示任务索引摘要，完整缓存控制归到“性能与缓存”，输入框外观归到“显示设置”。
- 将设置页改回顶部横向标签栏：恢复“发送规则 / 输入工具 / 记录设置 / 任务设置 / 新建文件模板库 / 筛选与侧栏 / 界面布局 / 显示设置 / 性能与缓存 / 高级设置”的胶囊标签样式，移除大卡片设置中心入口；“界面布局”仍保留桌面主页、侧边栏和移动端的图形化预览。
- 将“界面布局”设置页改为图形化布局设计器：顶部可切换桌面主页、侧边栏和移动端，中间显示可点击的模块预览，右侧显示当前区域设置；隐藏模块会以半透明虚线保留在预览中，可直接重新启用，并继续复用统一 `DisplayModule` 注册表和现有设置字段。
- 将设置页重组为卡片式“设置中心”：入口改为“发送与插入 / 输入工具 / 任务 / 新建文件模板库 / 界面布局 / 目录与筛选 / 性能与数据 / 高级设置”，并把界面布局拆成“桌面主页 / 侧边栏 / 移动端”；本次只移动和分组已有设置项，不改设置 key、默认值和功能逻辑。
- 新增统一显示模块系统第一阶段：建立 `DisplayModule` 注册表和 `homeLayout` / `sidebarLayout` / `mobileLayout` 三端显示配置；设置页新增“显示内容”标签，可分别选择完整、快速输入、导航、项目、任务、极简或自定义模式，并支持将主页/侧边栏/移动端显示配置一键同步，为后续主页、侧边栏和移动端共用同一套模块渲染打底。
- 统一修复任务插入格式：创建任务、任务按钮和发送到待办时会共用标准 `- [ ]` 任务行，清理 `- - [ ]`、`- * [ ]` 等重复列表符号；自定义父子任务模板会生成“父级时间戳任务 + 子级标准任务”，并保留优先级、开始/计划/截止/创建/完成日期和重复规则。
- 修复 Memos Plus 主页面移动端点击按钮会跳回输入框的严重交互问题：键盘避让和 `scrollIntoView` 只在主页面输入框真实聚焦时执行，点击外部按钮会取消待执行的输入框滚动；快速记录弹窗和侧边栏快速输入保持原有聚焦行为。
- 主界面搜索框右侧新增“Memos 设置”按钮，位于搜索框和刷新按钮之间，可直接打开 Memos Plus 插件设置页。
- 移动端性能模式新增“移动端主页布局”：可选择极简、只显示侧边栏、侧边栏 + 输入框、输入框 + 最近笔记、完整模式或自定义；自定义模式可逐项控制输入框、侧边栏、memo 列表、最近笔记、整理目录、任务目录、标签筛选、热力图、统计卡片、搜索框和刷新按钮，隐藏模块会跳过渲染和对应计算。
- 插入方式新增“新建标题后插入”：发送到文件/项目/模板时可输入新标题名称、选择标题级别、选择创建位置，并可设置同名标题时使用已有标题、仍然新建同名标题或取消；文件开头创建标题会避开 YAML frontmatter。
- 拆分任务筛选设置：新增“全库任务筛选”开关控制整理目录任务分支是否筛选整个 Obsidian 仓库，“任务索引缓存”单独负责加速全库任务筛选；任务索引状态新增缓存状态和全库筛选开关显示，并新增“清除任务筛选缓存”按钮。
- 新增全库 `TaskIndex`：在后台分批缓存 Markdown 任务行，左侧栏“整理 > 未完成任务”及优先级/到期子分支可统计和筛选整个 vault 的未完成任务；数量和点击筛选直接读缓存，设置页可启用/禁用、移动端延迟建立并手动重建索引。
- 新增发送失败草稿保护：普通保存或发送到项目/文件失败时，当前输入内容会保存在插件设置中，下次打开输入框自动恢复；设置页“记录设置”可关闭该能力。
- 新增移动端优先的轻量首页：iPhone 上默认显示紧凑输入框、稍后整理、收集箱和最近记录，并可一键切回完整工作台；设置页可关闭轻量首页、隐藏稍后整理按钮，并分别调整收集箱/最近记录的显示和高度。
- 左侧栏“整理 > 未完成任务”新增可展开子分支：可按 Obsidian Tasks 优先级筛选最高/高/中/低/最低/无优先级任务，并按截止日期筛选逾期、今日到期和本周到期；设置页可控制优先级分支、日期分支和默认展开状态。
- 限制侧边栏快速输入的 vault 预览读取量：`VaultSavedSearchIndex` 新增结果数和正文读取数上限，快速输入目录中的全库预览只读取当前预览所需的候选正文，避免纯正文/任务检索在侧边栏里扫完整个库。
- 优化 vault Saved Search 正文读取：当全库检索同时包含标签、状态、路径等 metadata 条件和正文/任务条件时，会先用共享 `VaultMetadataIndex` 缩小候选文件，再只读取候选文件正文，减少大仓库中不必要的 `vault.read`。
- 继续接入统一 `VaultMetadataIndex`：Saved Search 的全库检索现在对标签、状态、路径、日期、图片和链接等 metadata 条件复用共享索引；只有正文和任务条件才按需读取文件正文，主视图与侧边栏快速输入共用插件级索引。
