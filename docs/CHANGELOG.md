# Changelog

## 0.1.214 - Unreleased

- 修复输入框工具栏“新建 Excalidraw”的流程顺序：点击后现在会先进入 Memos Plus 的“选择文件 / 选择标题或插入位置”流程，用户确认目标后才检查并执行 Excalidraw 的嵌入命令；如果命令未找到，只在目标选择完成后提示启用 Excalidraw，不再一点击就跳过目标选择。

## 0.1.213 - Unreleased

- 修复移动端“选择文件 / 发送到文件”页面打开后需要先点一下才能操作的问题：打开 `MemosPlusMobilePanelView` 后会显式激活对应 leaf，并在渲染目标选择器后只聚焦面板根容器，不自动聚焦搜索框或唤起键盘，避免首次点击被 Obsidian 移动端用于激活页面而没有传给按钮、标签或搜索输入。

## 0.1.212 - Unreleased

- 继续收口移动端外观绘制成本：移动端输入框、浮动记录按钮、表格选择器、标签建议下拉和选择/模板标签页不再使用装饰性阴影、滤镜、渐变背景或过渡动画；设置页移动端布局预览也去掉热力图渐变和 sticky 面板阴影。移动端键盘弹出时输入框滚动改为即时定位，避免 smooth scroll 与 `visualViewport` 变化互相追逐；桌面端视觉层次和现有外观设置字段保持不变。

## 0.1.210 - Unreleased

- 优化移动端主页、侧边栏目录和快速目录列表项样式：移动端列表项统一使用 10px 低成本圆角变量，移除列表项阴影、滤镜、渐变背景和右侧更多按钮的额外圆角遮罩，避免出现内外双层圆弧叠加；桌面端继续保留较紧凑的原有视觉层级，不改变列表数据、点击、拖拽或筛选逻辑。

## 0.1.207 - Unreleased

- 提升移动端发送后稳定性：移动端输入框不再创建 Obsidian 内嵌 Markdown/CodeMirror editor，改用稳定 textarea 回退，避免 iOS 上 `cm-content` 聚焦、光标更新和发送后刷新链路触发 Obsidian Mobile 内部异常或插件重载；桌面端仍保留原生 Markdown editor 体验。
- 继续收口 Obsidian Community Promise warning：命令回调不再直接返回 async Promise，统一通过安全 helper 记录失败；主视图新增目录分组和移动端任务选项返回按钮等 UI 回调显式捕获异步错误，避免未处理 Promise 静默失败。

## 0.1.206 - Unreleased

- 收口视图激活和异步错误处理：主视图、侧边栏快速输入和移动端选择面板现在会等待 Obsidian `revealLeaf` 完成后再继续后续流程；主页/侧边栏自动填入剪贴板内容的 fire-and-forget Promise 增加错误记录，避免静默未处理；Excalidraw 嵌入命令执行失败时会提示用户确认插件状态。
- 优化自动检测剪贴板防重复自动填入：在 `clipboardAutoFill` 增加指纹去重和清除记忆链路（仅保存 `fingerprint`、`length`、时间戳、来源），避免同一段内容在同一会话与界面反复自动注入。
- 同一段剪贴板内容在“主页/侧边栏/移动端主页/快速记录”被自动填入后会被标记为 `auto-filled`，后续内容未变化时不会再次走自动检测，避免切换侧边栏、焦点与输入切换时重复触发。
- 当该内容由用户手动清空时，会记录 `dismissed`，直到剪贴板文本变化才允许再次自动填入；清空、长按粘贴、手动点击“填入/追加”仍保留原行为。
- 同步测试覆盖：`clipboardAutoFill` 与快速记录初始内容场景新增去重、清除恢复、链接内容重复拦截、隐私字段不存原文和移动端/手动粘贴分离用例，确保移动端不新增轮询、模态或焦点循环。

- 左侧栏新增大分组分割线，只在“全部笔记”和自定义目录区域之间显示，避免每个条目都被切开。
- 修复“发送到项目”弹窗项目、标签文件、搜索三个输入框输入时丢焦的问题：输入框不再触发整窗重绘，改为 200ms debounce 后只刷新对应结果列表，清空搜索会恢复全部结果。
- 快速记录弹窗改为复用主输入框的共享输入框/工具栏组件，支持同一套工具栏设置、更多菜单、Callout、代码块、Excalidraw、图片粘贴，以及发送到项目/默认位置流程。
- 输入框工具栏新增可配置的主工具栏/更多菜单，并加入代码块包装和 Excalidraw 快速创建；Excalidraw 会创建插件可识别的 `.excalidraw.md` 文件，并在输入框插入 `xxx.excalidraw` 链接。
- 修复“发送到项目”弹窗的“直接发送到默认位置”：现在复用普通 memo 保存流程，写入当前年份文件，不再发送到项目或项目分类。

- 修复发送弹窗顶部 `+` 按钮无响应：改用 Obsidian 原生输入弹窗添加自定义标签页，兼容桌面端和移动端。
- memo 右侧“更多操作”菜单新增“转到项目”，并可在设置中选择投递成功后保留、归档或删除原 memo；取消投递或投递失败时不会修改原 memo。
